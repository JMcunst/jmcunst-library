# 코드 리뷰 실천 (Code Review Practice)

## 정의

코드 변경을 병합 전에 동료가 검토해 결함, 보안 리스크, 설계 문제를 사전에 줄이고 팀 지식을 공유하는 운영 절차.

## 왜 중요한가?

- 운영 장애의 많은 원인을 배포 전에 차단할 수 있다.
- 특정 개인에게 집중된 코드 맥락을 팀 지식으로 전환한다.
- 품질 기준을 코드베이스 전체에 일관되게 적용할 수 있다.

## 책 내용 핵심 정리

- 코드 리뷰는 장기적으로 시스템을 견고하게 만드는 핵심 기본기다.

## 실무 포인트

### 1. 리뷰 가능한 크기 유지
- PR 크기를 작게 유지해 리뷰 정확도를 높인다.
- 기능/리팩터링/포맷 변경은 가능한 한 분리한다.

### 2. 리뷰 기준 명확화
- 기능 정확성, 실패 시나리오, 보안, 운영 영향(로그/메트릭)을 기본 체크 항목으로 둔다.
- "취향"보다 "리스크" 중심 피드백을 우선한다.

### 3. 머지 게이트 자동화
- 필수 리뷰어 수와 필수 CI 통과 조건을 브랜치 보호 규칙으로 강제한다.
- 핫픽스 예외는 사후 리뷰 절차를 갖춘다.

## 바로 적용 가능한 체크리스트

- [ ] PR 템플릿에 변경 의도/영향/롤백 계획을 포함했다.
- [ ] 필수 리뷰어 정책을 저장소에 적용했다.
- [ ] CI 실패 상태에서는 병합할 수 없게 설정했다.
- [ ] 리뷰 코멘트 누락 없이 해결/응답 프로세스를 운영한다.

## 실무 한 줄

좋은 코드 리뷰는 코드 품질 개선이 아니라, 운영 리스크를 병합 전에 제거하는 과정이다.

## 관련 개념

- Pull Request workflow
- Branch protection
- Static analysis
- Secure coding review
