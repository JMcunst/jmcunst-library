# 기본 컴포넌트 활용 (Core Component Usage Basics)

## 정의

쿠버네티스 기본 리소스의 역할을 섞지 않고, 워크로드 성격에 맞게 정확히 선택해 조합하는 운영 기본기.

## 왜 중요한가?

- 리소스를 잘못 선택하면 배포는 되지만 운영이 불안정해진다.
- 규모가 커질수록 "예외 설계"가 누적되어 운영 복잡도가 폭증한다.
- 표준 사용 패턴이 있어야 팀 간 인수인계와 자동화가 쉬워진다.

## 책 내용 핵심 정리

- 기본 컴포넌트를 어떻게 사용하는지 이해하는 것이 중요하다.
- 고급 주제를 다루더라도 기본기 위에서만 확장 가능하다.

## 실무 포인트

### 1. 리소스 선택 규칙
- Stateless 앱은 Deployment, Stateful 앱은 StatefulSet으로 구분한다.
- 배치성 작업은 Job/CronJob으로 분리한다.

### 2. 설정과 비밀값 분리
- 일반 설정은 ConfigMap, 자격증명은 Secret으로 관리한다.
- 런타임 주입 방식을 표준화해 환경 차이로 인한 오류를 줄인다.

### 3. 네트워크/스토리지 기본 원칙
- 서비스 디스커버리는 Service를 기본으로 사용한다.
- 영속 데이터는 PVC를 통해 다루고, 스토리지 클래스를 명시한다.

## 바로 적용 가능한 체크리스트

- [ ] 워크로드별 리소스 선택 기준표를 문서화했다.
- [ ] 템플릿 생성 시 기본 리소스 조합을 고정했다.
- [ ] ConfigMap/Secret 혼용 없이 분리했다.
- [ ] Stateful 워크로드는 복구 전략을 함께 설계했다.

## 실무 한 줄

기본 컴포넌트를 정확히 쓰는 팀은 장애 때 빠르고, 성장할수록 더 단단해진다.

## 관련 개념

- Deployment / StatefulSet
- Job / CronJob
- ConfigMap / Secret
- Service / PVC
