# 지속적 배포 실천 (Continuous Delivery Practice)

## 정의

빌드, 테스트, 보안 검증, 배포를 자동화해 작은 변경을 자주, 안전하게 운영 환경에 반영하는 배포 방식.

## 왜 중요한가?

- 변경 배치 크기를 줄여 실패 시 영향 범위를 최소화한다.
- 배포 속도와 안정성을 동시에 개선할 수 있다.
- 장애 복구 시간을 단축하고 운영 신뢰도를 높인다.

## 책 내용 핵심 정리

- 지속적인 서비스 배포 역량은 쿠버네티스 활용의 기본기이자 확장 기반이다.

## 실무 포인트

### 1. 파이프라인 게이트 명확화
- 단위/통합 테스트, 취약점 스캔, 정책 검증을 배포 전 필수 게이트로 둔다.
- 환경별 배포 승인 기준(자동/수동)을 명확히 구분한다.

### 2. 점진적 릴리스 전략
- 블루그린, 카나리, 점진 롤아웃을 서비스 특성에 맞게 선택한다.
- 배포 중 핵심 메트릭 에러율/지연시간을 실시간 감시한다.

### 3. 롤백 자동화
- 배포 실패 조건을 정의하고 자동 롤백 기준을 설정한다.
- 롤백 절차를 정기적으로 리허설한다.

## 바로 적용 가능한 체크리스트

- [ ] CI/CD 파이프라인이 코드 저장소와 연결되어 있다.
- [ ] 배포 전 품질/보안 게이트를 자동 검증한다.
- [ ] 카나리 또는 블루그린 중 하나를 표준 배포 전략으로 운영한다.
- [ ] 롤백 절차와 임계치 기반 알림을 구성했다.

## 실무 한 줄

지속적 배포의 목표는 "빨리 배포"가 아니라, "실패해도 빨리 회복"이다.

## 관련 개념

- CI/CD pipeline
- Canary / Blue-Green
- Progressive delivery
- Rollback automation
