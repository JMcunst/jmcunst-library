# 버전 관리 실천 (Version Control Practice)

## 정의

코드와 설정 변경 이력을 추적 가능하게 관리해, 릴리스 근거와 롤백 기준을 명확히 만드는 개발/운영 기본 체계.

## 왜 중요한가?

- 어떤 변경이 어떤 장애를 만들었는지 역추적할 수 있다.
- 협업 시 충돌을 줄이고 변경 의도를 팀이 공유할 수 있다.
- 배포 파이프라인의 재현성과 감사 가능성을 확보한다.

## 책 내용 핵심 정리

- 버전 관리를 포함한 기본기를 잘 닦아야 견고한 시스템을 만들 수 있다.

## 실무 포인트

### 1. 변경 단위 작게 유지
- 하나의 커밋은 하나의 의도만 담는다.
- 큰 변경은 단계적으로 나눠 위험을 분산한다.

### 2. 태그/릴리스 추적성
- 릴리스마다 Git 태그를 남기고 변경 로그를 연결한다.
- 이미지 태그, 배포 리비전, Git SHA를 연계해 추적한다.

### 3. 브랜치 전략 일관성
- trunk-based 또는 짧은 feature branch 전략을 팀 단위로 통일한다.
- 오래된 장수 브랜치는 정기 정리한다.

## 바로 적용 가능한 체크리스트

- [ ] 릴리스마다 Git 태그를 발행한다.
- [ ] PR 단위 변경 범위를 제한한다.
- [ ] 배포 아티팩트에 Git SHA를 포함한다.
- [ ] 롤백 기준 커밋/태그를 운영 문서에 명시한다.

## 실무 한 줄

버전 관리는 협업 도구이면서 동시에 장애 복구 도구다.
