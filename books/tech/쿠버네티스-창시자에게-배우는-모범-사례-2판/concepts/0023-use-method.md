# USE 방법론 (USE Method)

## 정의

브렌던 그레그가 제안한 인프라 관측 프레임워크로, 모든 자원에 대해 사용률(Utilization), 포화도(Saturation), 에러(Errors)를 확인하는 방법.

## 왜 중요한가?

- 리소스 병목을 구조적으로 진단할 수 있다.
- CPU/메모리/디스크/네트워크 등 인프라 계층 장애를 빠르게 좁힐 수 있다.
- 복잡한 분산 시스템에서도 공통 기준으로 적용 가능하다.

## 책 내용 핵심 정리

- USE는 분산 시스템 모니터링의 대표 패턴 중 하나다.
- 애플리케이션 수준보다는 인프라 수준 모니터링에 더 적합하다.
- 네트워크 병목 같은 리소스 제약을 신속하게 파악하는 데 강점이 있다.

## 실무 포인트

### 1. U: 사용률
- 현재 자원이 얼마나 사용 중인지 본다.
- 예: CPU 사용률, 디스크 I/O 사용률

### 2. S: 포화도
- 대기열/병목으로 인해 처리 대기 중인 수요를 본다.
- 예: run queue 길이, 디스크 큐 길이

### 3. E: 에러
- 리소스 계층 오류를 본다.
- 예: 네트워크 drop/error, 디스크 오류, OOM

## 바로 적용 가능한 체크리스트

- [ ] 노드/네트워크/스토리지에 USE 지표를 매핑했다.
- [ ] 사용률만 보지 않고 포화도 지표를 함께 본다.
- [ ] 에러 지표 임계치를 서비스 중요도별로 다르게 둔다.
- [ ] 병목 의심 시 USE 순서(U->S->E)로 점검한다.

## 실무 한 줄

USE는 "느리다"는 증상을 "어느 자원이 막혔는가"로 번역해주는 인프라 진단 언어다.

## 관련 개념

- Saturation

- Resource bottleneck
